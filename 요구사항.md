나는 사람들이 사진과 글을 업로드하고, 사이트에서 온라인 전시할 수 있는 형태의 서비스를 만드려고해. 나는 코딩을 인생에 있어서 한 번도 해본 적이 없는 사람이라 사용 방법까지 정리해줘. python은 설치했어. (3.14.5 버전이야)

내가 필요한 요구사항은 아래와 같아.

1. 프로젝트 개요

1.1 서비스명: 나의 반려쓰레기 해방일지 서비스

1.2 한 줄 소개: 버리고싶었지만 그동안 버리지 못했던 반려쓰레기를 기록하고, 전시해주는 형태의 플랫폼

1.3 대상 사용자: 버리지 못해서 고민인 사람들 모두 (미련이 있지만 이제는 버려야지 생각했을때 찾아오는 곳이야)

1.4 주요 화면 : 아카이브(전시장), 임시저장소, 해방일지 기록하기 등

- 아카이브(전시장) : 사람들이 업로드한 이미지를 웹매거진처럼 멋지게 보여주는 화면
- 로그인, 회원가입화면
- 해방일지 기록하기 : 콘텐츠 작성 화면
- 임시저장소 아카이브
- 실물 전시 참여 물품 목록 아카이브

1.5 사용 흐름: 로그인 → 영상 목록에서 원하는 강의 클릭 → 재생 페이지에서 시청

2. 기술 스택: Django 모놀리식

3. 요구사항:

- 콘텐츠 이미지를 클릭하거나 마우스를 가져다대면 제목 또는 간단한 스토리를 볼 수 있음
- 자세한 히스토리를 보려면 회원가입 후 로그인 해야 함
- 콘텐츠를 작성할 때는 간단하게 닉네임과 수정을 위한 비밀번호를 입력받기
- 콘텐츠를 작성할 때는 즉시해방-직접 버리기, 즉시해방 - 실물전시에 참여하기(기부됨), 임시보관하기
- 콘텐츠를 보는 사람은 댓글과 공감, 슬픔, 화남의 이모지 반응을 남길 수 있도록
- 댓글은 익명으로 보이되, 회원가입을 한 사람만 작성할수 있도록
- 콘텐츠는 누구나 올릴 수 있지만 업로드 할 때는 관리자의 승인이 필요
- 회원가입 후 콘텐츠를 올리는 등의 활동을 하는 사람에게는 포인트를 지급하고싶어
- 임시저장소에 올라온 콘텐츠는 관리자가 인스타그램 스토리에 올릴거야. 사람들의 판결을 받는거지! 사이트 자체에서도 버리라, 말라 의견을 남길수 있도록 하면 좋을것같다.
- 실물전시 참여예정 물품에 올라온 콘텐츠는 회원들의 찜하기 기능을 넣고싶어. 찜은 3명까지만 가능하도록. 여기에 모인 물품들은 나중에 오프라인전시회에에서 판매해서 수익금을 기부하거나 물품 자체를 기부하려고해. 찜한 사람들에게 구매우선권을 주는 아이디어를 생각하고 있어.
- 회원가입한 사람들은 프로젝트멤버, 회원 개념으로 구분하고 프로젝트 멤버는 이번 기수 프로젝트에 규칙적으로 참여하는 사람이야
- 전시가 확정되면 펀딩을 통해 바우처기능이 있는 입장권을 판매할거야. 이 바우처로 실물전시되는 물건들과 제작되는 도감을 구입할 수 있도록 기획하고 있으니 이점도 참고해줘
- 콘텐츠를 아카이브에 올릴때는 흰 배경에 단정히 올리고, 정면에서 촬영하길 요구하고 싶어. 함께 쓰는 텍스트는 에세이 형태로 작성하도록 가이드하고 싶어.
- 멋진 UI면 좋겠어.
- 배포는 render에서 할 거야. 그래서 requirements.txt 파일이 필요하다네! render 서비스에서 배포할 때 처음에 `gunicorn 프로젝트명.wsgi`으로 시작한데! 참고해줘.
